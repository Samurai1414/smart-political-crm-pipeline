02 · Complete Complaint Lifecycle Flow
📝

Step 1 · Citizen Submits Complaint

CITIZEN

Citizen form bharta hai: Naam, Phone, Area, Complaint Text (Hindi/English/Hinglish). Optional mein ek ya zyada BEFORE photos attach kar sakta hai — jaise tooti hui sadak ki photo, bhari hui gutter ki photo. Ye evidence ke taur pe kaam karta hai.

📸 Before Photo (optional) 📍 Location / Ward 📞 Phone Number 🤖 ML Auto-categorizes
👁️

Step 2 · Admin Reviews & Assigns

DEPT ADMIN

Department Admin ke dashboard pe complaint aati hai — category, urgency, sentiment score ke saath. Admin citizen ki before-photo dekh sakta hai. Admin complaint ek Field Worker ko assign karta hai aur deadline set karta hai.

⚡ Urgency visible 📸 Before photo review 👷 Assign worker 📅 Set deadline
🔔

Step 3 · Worker Gets Notified

FIELD WORKER

Field Worker ko in-app notification milti hai: "Tumhe ek naya kaam assign hua hai." Worker apne dashboard pe sirf apne assigned complaints dekhta hai. Complaint details, citizen ki before-photo, aur location address dikhta hai.

📲 In-app notification 📍 Address + area 📸 Before photo visible
🔧

Step 4 · Work Done → Photo Upload

FIELD WORKER

Worker kaam complete karta hai. Worker apne phone se AFTER photo kheechta hai aur dashboard pe upload karta hai. Ek chhoti note bhi likh sakta hai: "Pipe repair kar diya gaya, paani ab aa raha hai." Status "Work Done — Pending Approval" ho jata hai.

📸 After photo upload 📝 Work notes 🕐 Timestamp auto-save ⏳ Pending Approval
🔍

Step 5 · Admin Reviews Before + After Photos

DEPT ADMIN

Admin ko notification milti hai. Admin dashboard pe side-by-side Before vs After photo compare karta hai. Worker ki notes bhi dikhti hain. Admin decide karta hai: APPROVE ya REJECT.

🖼️ Before vs After compare 📝 Worker notes ✅ Approve button ❌ Reject button
↓ Admin ka decision
✅

Step 6A · APPROVED → Citizen Notified

RESOLVED

Admin approve karta hai → Complaint status "Resolved ✅" ho jati hai. Citizen ke dashboard pe notification aati hai: "Aapki shikayat #4821 resolve ho gayi hai." Citizen AFTER photo dekh sakta hai. Citizen ke paas Satisfied / Not Satisfied button hota hai.

✅ Status: Resolved 📲 Citizen notified 📸 After photo visible 👍 Satisfaction feedback
🔄

Step 6B · NOT SATISFIED → Re-Raise

RE-OPENED

Agar admin reject karta hai ya citizen "Not Satisfied" click karta hai → Citizen ek Re-raise form bharta hai: reason likhta hai + naya photo attach kar sakta hai. Complaint wapas Step 2 pe chali jaati hai with HIGHER priority. Sab purani history preserve hoti hai.

🔄 Re-raised complaint ⬆️ Higher priority 📸 New photo + reason 📋 Full history saved
03 · Photo Upload System — Kaise Kaam Karega

📸 Evidence Photo System

Citizen uploads BEFORE photo · Worker uploads AFTER photo · Admin compares both

🧑‍💼 Citizen — Before Photo

  • Form mein <input type="file"> field hoga
  • Max 3 photos, max 5MB each (JPEG/PNG)
  • Frontend: FileReader API se preview dikhega
  • Backend: FastAPI mein UploadFile handle karega
  • Storage: Server pe /uploads/before/ folder mein save
  • DB mein photo path save hoga (string field)

👷 Worker — After Photo

  • Worker dashboard pe "Upload Completion Photo" button
  • Mobile-friendly — directly phone camera se upload
  • Backend: /uploads/after/ folder mein save
  • Admin dashboard pe side-by-side comparison dikhega
  • Timestamp automatically save hogi with photo
  • Agar kaam nahi hua — worker notes mandatory

06 · ML Engine — Lifecycle mein kahan kaam karta hai

🤖 ML Kab aur Kahan Trigger Hota Hai?

Citizen submit karte hi — background mein 4 models ek saath chalte hain. Koi wait nahi.

🗂️ Categorization Complaint text → Water / Roads / Electricity / Health / Security / Other category decide karta hai. Isi se Admin ko pata chalta hai kaunse department ko assign karein. FastText
😤 Sentiment Text ka mood detect karta hai — Positive / Negative / Neutral + Anger Score. High anger = complaint red mein dikhegi Admin dashboard pe. VADER + mBERT
⚡ Urgency Score Sentiment + keywords ("3 din se", "bahut problem", "emergency") se HIGH / MEDIUM / LOW urgency decide hoti hai. HIGH complaints sabse upar dikhti hain. Rule-based
🔔 Notification Category se best-match template select hoti hai. Citizen ka naam, complaint ID, department, ETA automatically fill hota hai. Hindi ya English — citizen ki preference se. TF-IDF
🗂️

Model 1 · FastText

Complaint Categorization

KYA KARTA HAI → Text padh ke category decide karta hai
KYUN FASTTEXT → Hinglish ke liye best, subword embeddings se "paani", "pani", "water" sab samajhta hai
TRAINING → 200 labelled complaints kaafi hain demo ke liye
OUTPUT → label: "Water" + confidence: 0.93
INSTALL → pip install fasttext
😤

Model 2 · VADER + mBERT

Sentiment Analysis (Dual Model)

KYA KARTA HAI → Text ka mood: Positive / Negative / Neutral
VADER → English text ke liye — fast, no training needed
mBERT → Hindi + Hinglish ke liye — 104 languages pretrained
OUTPUT → sentiment: "negative" + anger_score: 0.78
INSTALL → pip install vaderSentiment transformers
📍

Model 3 · XGBoost

Predictive Hotspot Analytics

KYA KARTA HAI → Kaun sa area agle 7 din mein surge dekhega
INPUT → category + ward/area + timestamp + frequency
OUTPUT → "Ward 14 mein paani ki complaints badhne wali hain"
MAP → Folium se interactive hotspot map banta hai
INSTALL → pip install xgboost geopandas folium
🔔

Model 4 · TF-IDF

Smart Notification Engine

KYA KARTA HAI → Complaint ke liye best message template choose karta hai
KAISE → TF-IDF vectors banata hai, cosine similarity se best match dhundhta hai
OUTPUT → "आपकी शिकायत #4821 जल बोर्ड को दी गई। 48 घंटे में हल होगा।"
INSTALL → pip install scikit-learn (already in sklearn)

🇮🇳 Hinglish Preprocessing Pipeline

Raw Text
"paani nahi aa raha" → langdetect
lang = "hinglish" → Transliterate
Roman→Devanagari → IndicNLP Clean
stopwords remove → 4 Models
parallel inference
▸ indic-transliterate — Romanized Hindi ko Devanagari mein convert karta hai
▸ IndicNLP / iNLTK — Hindi tokenization aur stopword removal
▸ langdetect + custom rules — Mixed script Hinglish ko detect karta hai
▸ Install: pip install langdetect indic-transliterate inltk indicnlp
