Clean and normalize text: remove noise, transliterate Hinglish to Hindi/English, tokenize, remove stopwords.

IndicNLP / iNLTK indic-transliterate regex NLTK

Preprocessed text is passed simultaneously to all 4 ML models. Each returns a structured output that is merged into one unified complaint object.

FastText mBERT VADER XGBoost TF-IDF Parallel Inference

02 · The 4 ML Models
🗂️

Complaint Categorization

FastText · Text Classification
Classifies complaint into: Water, Roads, Electricity, Health, Security, Corruption, Other.

FastText handles Hinglish well via subword embeddings. Train on labelled complaint dataset (~1000 examples sufficient for demo).
😤

Sentiment Analysis

VADER + mBERT · Dual Model
VADER handles English sentences fast. mBERT (multilingual-BERT) handles Hindi + Hinglish sentences.

Output: Positive / Negative / Neutral + anger score. High-anger complaints flagged as URGENT.
📍

Predictive Hotspot

XGBoost + GeoPandas · Trend Model
Takes: complaint category, location (ward/area), timestamp, frequency. Predicts: which areas will have surge in complaints next 7 days.

Map visualization using Folium.
🔔

Smart Notification Engine

TF-IDF + Cosine Similarity
Matches complaint type → pre-written response templates. Selects best-fit template. Fills in citizen name, complaint ID, status, ETA.

Sends in citizen's preferred language.

04 · Hinglish-Specific Strategy
🇮🇳

Hindi + English + Hinglish Support

  • Use langdetect + custom rules to identify Hinglish (mixed script / romanized Hindi)
  • Use indic-transliterate library to convert Romanized Hindi → Devanagari before model input
  • FastText subword embeddings handle OOV Hinglish words natively
  • mBERT is pretrained on 104 languages including Hindi — fine-tune on your Hinglish dataset
  • VADER used only for English-dominant sentences; mBERT fallback for rest
  • Notification templates available in both Hindi and English, selected by citizen preference
